# מודל הביצוע — "רץ כמו אמיתי, מתחלף בלי לשנות מתכונת"

> המטרה: הנייר מדמה מסחר אמיתי לפרטי-פרטים, והמעבר ללייב הוא **החלפת שכבת
> ביצוע אחת** — כל שאר הבוט (אותות, סייזינג, סולם, טריילינג, מגנים, רוטציה)
> נשאר בדיוק אותו קוד. המסמך הזה מוכיח את שני החצאים.

## חלק 1 — מה הנייר כבר מדמה מהמציאות (v54)

| מנגנון אמיתי | איך הנייר מדמה | היכן בקוד |
|---|---|---|
| עמלת taker (כניסה/סטופ/timeout) | `FEE=0.0005`/צד על כל market fill | לאורך כל entries/exits |
| עמלת maker (שלבי סולם 0.6R/1.0R) | `FEE_MAKER=0.0002`, מילוי במחיר הרמה | LADDER_06 / LADDER_10 |
| Slippage (מילוי שוק גרוע מהמחיר) | `SLIP=0.0003` (3bps) על כל כניסה/סטופ | `fillPx4`, `exitPx`, ROTA fills |
| Funding פרפ (החזקת פוזיציה) | `FUND_8H=0.0001`, שעתי, לונג משלם/שורט מקבל | בלוק `utcM===0` |
| Mark-to-market (שווי חי) | ההון נמדד לפי מחיר השוק, לא הכניסה | בלוק `bot_equity` |
| נעילת maker נקייה | שלבי הסולם ממלאים במחיר המדויק (limit) | ללא SLIP על maker |
| מינימום גודל פוזיציה | DONCH4H ≥$500, ROTA ≥1% מהתיק | בדיקות notional |

**מסקנה**: עקומת ההון, ה-checkpoint וכל הסטטיסטיקות כבר משקפות כלכלת פרפ
אמיתית. מה שתראה בנייר ≈ מה שתראה בלייב (עד פערי מילוי אמיתיים, ראה חלק 3).

## חלק 2 — "תפר הביצוע": מה בדיוק מתחלף בלייב

הבוט בנוי בשתי שכבות שכבר מופרדות:
- **שכבת אסטרטגיה** (95% מהקוד): מתי להיכנס, כמה, איפה הסטופ, ניהול הסולם,
  טריילינג, מגנים, רוטציה. **לא משתנה כלל בלייב.**
- **שכבת ביצוע** (~5 נקודות): הרגע שבו "פקודה ממומשת". היום = INSERT/UPDATE
  ל-`bot_trades` במחיר-מילוי מדומה. בלייב = קריאה למתאם-בורסה שמניח פקודה
  אמיתית וכותב בחזרה את המילוי האמיתי.

**5 נקודות התפר (כאן ורק כאן משתלב מתאם-בורסה):**
1. פתיחת DONCH4H — `insert bot_trades ... strategy:'DONCH4H'`
2. פתיחת ROTA — `insert bot_trades ... strategy:'ROTA'`
3. יציאות סולם (LADDER_06 / LADDER_10) — limit orders
4. סגירה סופית / סטופ / timeout — market order
5. סגירת/פתיחת רוטציה בריבאלנס

**מנגנון ההחלפה**: הדגל `bot_state.paper_mode` כבר קיים ומוטבע על כל טרייד
(`paper_mode: paperMode`). בלייב: `paper_mode=false` + מתאם-בורסה נוכח →
5 הנקודות מנתבות לבורסה במקום ל-DB-בלבד. **אפס שינוי בשכבת האסטרטגיה.**

## חלק 3 — מה עדיין ייבדק רק בלייב (הפער שנסגר בהון קטן)
- **מילוי maker אמיתי**: הנייר מניח ששלב-סולם ממלא בנגיעה; בלייב limit *עשוי*
  לא להתמלא → נפילה ל-taker. הפער קטן, נמדד בהון קטן.
- **Slippage אמיתי** על אלטים דקים מול ההנחה 3bps.
- **Lot-size / min-notional** של הבורסה הספציפית (עיגול כמויות).
- **דחיות פקודה / rate-limit** — המתאם יטפל ב-retry + reconciliation.

## חלק 4 — צ'קליסט המעבר (הכל מוכן חוץ מהמתאם עצמו)
- [x] כלכלת פרפ מדומה מלאה (חלק 1)
- [x] תפר ביצוע מבודד ומתועד (חלק 2)
- [x] דגל `paper_mode` מוטבע על כל טרייד
- [x] מגני סיכון פועלים זהה בשני המצבים
- [ ] **מתאם-בורסה** (5 נקודות התפר) ← שלב המשתמש, נפרד
- [ ] position reconciliation (בורסה מול DB) — חלק מהמתאם
- [ ] הרצת הון קטן ($500-1000) לאימות הפער בחלק 3

## עקרון-על
הבוט לא "יעבור לאחר" מסחר אמיתי — הוא **כבר מריץ** את אותה מתכונת בדיוק, רק
שהמילוי מדומה במקום אמיתי. חיבור הבורסה = להחליף את שכבת המילוי ב-5 נקודות
מתועדות. שום אסטרטגיה, שום סייזינג, שום מגן — לא נוגעים. זה מה שאתה מבקש:
"רץ כמו אמיתי, מתחבר בלי לשנות מתכונת."
