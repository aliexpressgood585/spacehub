# מעבר למסחר אמיתי — הצעדים שלך (v55)

> המתאם לבורסה (Bybit) כבר בקוד, נעול בשלושה מנעולים. שום דבר לא זז עד
> שתשלים את הצעדים כאן. **לעולם אל תשלח מפתחות API בצ'אט — לאף אחד.**

## שלב 1 — חשבון Bybit (שלך בלבד, ~30 דקות)
1. פתח חשבון ב-bybit.com ועבור אימות זהות (KYC level 1).
2. הפקד את הון ההתחלה. **המלצה מתועדת: $500-1,000 בלבד** לשלב האימות —
   מספיק למדוד slippage ומילויים אמיתיים; טעות לא כואבת. הגדלה רק אחרי
   צ'קפוינט 50 העסקאות והשוואת נייר-מול-אמיתי.
3. ודא שהכסף בחשבון **Unified Trading** (לא Funding).

## שלב 2 — מפתח API (הגדרות קריטיות!)
ב-Bybit: פרופיל → API → Create New Key → System-generated:
- הרשאות: **Read-Write ל-Trade בלבד** (Contract — Orders & Positions).
- **בלי הרשאת Withdrawal. בלי Transfer. לעולם.**
- IP restriction: השאר ללא הגבלה (ה-IP של Supabase משתנה) — זו בדיוק
  הסיבה שאסור הרשאת משיכה.
- שמור את ה-Key וה-Secret בצד — לא בצ'אט, לא בקובץ בריפו.

## שלב 3 — הזנת המפתחות (בממשק GitHub, לא בצ'אט)
בריפו: Settings → Secrets and variables → Actions:
1. **New repository secret**: שם `BYBIT_API_KEY`, ערך = ה-Key.
2. **New repository secret**: שם `BYBIT_API_SECRET`, ערך = ה-Secret.
3. לשונית **Variables** → New repository variable: שם `LIVE_TRADING`,
   ערך `0` (עדיין כבוי!).

## שלב 4 — חימוש (כשתחליט, לא לפני)
1. שנה את המשתנה `LIVE_TRADING` מ-`0` ל-`1`.
2. הרץ את ה-workflow ‏Deploy Edge Function (או בקש ממני).
3. בדשבורד: כבה מצב נייר (הכפתור "נייר" → "אמיתי").
4. משלב זה הבוט מסחר אמיתית. ה-reconciliation משווה בורסה-מול-DB כל
   5 דקות ומתריע על כל פער; כיבוי מיידי = להחזיר את הכפתור ל"נייר".

## מה קורה כשהמנעולים סגורים (עכשיו)
כלום. אפס שינוי בהתנהגות. הבוט ממשיך נייר בדיוק כמו היום.

## פערים ידועים שנמדדים בהון הקטן (לפני הגדלה)
- שלבי הסולם מבוצעים ב-v55 כ-market (taker) ולא limit (maker) — פער
  ~3bps לרגל; ישודרג ל-resting limits אחרי אימות.
- slippage אמיתי מול הנחת ה-3bps.
- מילוי מינימלי/עיגול לוט על אלטים קטנים (מטופל, נמדד).
